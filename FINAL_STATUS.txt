HOW TO RUN THE PROJECT
=================================================================

OPTION 1: Run Everything with Docker
=====================================
cd PointAtlas

# Start all services
docker compose up -d

# View logs
docker compose logs -f

# Access:
# - Frontend: http://localhost:3000
# - Backend API: http://localhost:5000
# - Swagger: http://localhost:5000/swagger

# Default admin credentials:
# Email: admin@pointatlas.com
# Password: Admin@123456


OPTION 2: Run Backend & Frontend Separately
=============================================
# Terminal 1 - Start PostgreSQL
docker run -d --name pointatlas-postgres \
  -e POSTGRES_DB=pointatlas \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -p 5432:5432 \
  postgis/postgis:16-3.4

# Terminal 2 - Run Backend
cd backend/PointAtlas.API
dotnet run

# Terminal 3 - Run Frontend
cd frontend
npm run dev

# Access:
# - Frontend: http://localhost:5173
# - Backend: http://localhost:5000


OPTION 3: Test Backend Only (No Frontend)
==========================================
# Start database
docker run -d --name pointatlas-postgres \
  -e POSTGRES_DB=pointatlas \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -p 5432:5432 \
  postgis/postgis:16-3.4

# Run backend
cd backend/PointAtlas.API
dotnet run

# Open Swagger UI in browser
# http://localhost:5000/swagger

# Test with admin credentials:
# Email: admin@pointatlas.com
# Password: Admin@123456

TESTING THE APPLICATION
=================================================================

1. Register a New User:
   - Go to http://localhost:3000/register
   - Fill in the form
   - Password must: 8+ chars, uppercase, lowercase, digit, special char

2. Login:
   - Go to http://localhost:3000/login
   - Use your credentials or admin credentials

3. View Markers:
   - You'll see the map with any existing markers
   - Pan and zoom the map
   - Markers auto-filter based on map bounds

4. Add Markers:
   - Click "Add Marker" button, OR
   - Click anywhere on the map
   - Fill in the form
   - Click "Create Marker"

5. Search and Filter:
   - Use search box to search marker titles/descriptions
   - Use category dropdown to filter by category
   - Filters update automatically

6. View Marker Details:
   - Click on any marker
   - Popup shows: title, description, category, author
   - If you own the marker or are admin, Edit/Delete buttons appear

Clean Architecture:
- Core (Domain) - No dependencies
- Infrastructure (Data Access) - Depends on Core
- Application (Business Logic) - Depends on Core
- API (Presentation) - Depends on Application & Infrastructure

- Service Layer Pattern
- DTO Pattern
- Dependency Injection
- Context API (React)
- Protected Routes
